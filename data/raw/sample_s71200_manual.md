# S7-1200 PLC 维护手册(合成测试数据)

> ⚠️ 本文件是端到端冒烟测试用的合成数据,内容参考公开常识编写,不是西门子官方文档。
> 真实项目请到西门子工业支持中心下载官方手册后删除本文件。

## 第一章 CPU 状态指示灯

S7-1200 CPU 正面有三个状态指示灯:RUN/STOP、ERROR、MAINT。

RUN/STOP 灯为绿色常亮时,表示 CPU 处于运行模式,用户程序正在执行。
RUN/STOP 灯为黄色常亮时,表示 CPU 处于停止模式,用户程序未执行,此时可以下载程序。
RUN/STOP 灯绿色与黄色交替闪烁时,表示 CPU 正在启动过程中。

ERROR 灯红色闪烁表示存在可诊断的错误,例如程序错误、组态错误或 I/O 故障,
可以通过 TIA Portal 的在线诊断功能读取具体错误信息。
ERROR 灯红色常亮表示 CPU 存在严重内部错误或硬件故障,建议执行以下步骤:
1. 断电重启 CPU;
2. 如果故障依旧,拔出存储卡检查是否损坏;
3. 使用 TIA Portal 读取诊断缓冲区;
4. 仍无法恢复时执行恢复出厂设置,若故障仍存在则需要更换 CPU 模块。

MAINT 灯黄色常亮表示需要维护,常见原因包括存储卡即将写满或硬件即将到达维护周期。

## 第二章 以太网通信配置

S7-1200 通过集成的 PROFINET 接口进行以太网通信。设置 IP 地址的步骤:
1. 在 TIA Portal 项目树中打开"设备组态";
2. 选中 CPU 的 PROFINET 接口,在属性窗口中选择"以太网地址";
3. 输入 IP 地址和子网掩码,例如 192.168.0.1 / 255.255.255.0;
4. 编译并下载硬件组态到 CPU。

通信故障排查:如果编程电脑无法连接 CPU,依次检查:
1. 网线连接和交换机端口指示灯;
2. 电脑网卡 IP 是否与 CPU 处于同一网段;
3. 使用 TIA Portal 的"可访问设备"功能扫描网络;
4. 防火墙是否阻止了 TIA Portal 的通信端口;
5. 如果扫描不到设备,尝试直连(不经过交换机)排除交换机故障。

## 第三章 程序下载失败的处理

程序下载失败的常见原因和处理方法:
1. CPU 处于 RUN 模式且下载内容包含硬件组态 — 需要将 CPU 切换到 STOP 模式;
2. 存储卡写保护 — 检查存储卡侧面的写保护开关;
3. 程序块受专有技术保护且密码不正确 — 联系程序原作者获取密码;
4. 通信中断 — 参考第二章通信故障排查;
5. CPU 固件版本低于项目组态的版本 — 升级 CPU 固件或在项目中修改设备版本。

## 第四章 存储卡的使用

S7-1200 使用 SIMATIC 存储卡(SMC)。存储卡有三种工作模式:
程序卡模式:程序保存在卡中,拔卡后 CPU 无法运行。
传送卡模式:上电时把卡中程序复制到 CPU 内部存储器,复制完成后可拔卡。
固件更新卡模式:用于升级 CPU 固件版本。

存储卡故障的典型现象是 CPU 启动后 ERROR 灯常亮或程序丢失。
处理方法:用读卡器在电脑上检查卡内文件结构是否完整,必要时重新格式化
(注意:必须使用 TIA Portal 或专用工具格式化,不能用 Windows 直接格式化)。

## 第五章 模拟量模块接线与故障

模拟量输入模块 SM1231 支持电压型和电流型信号。
电流型(4-20mA)接线时注意二线制与四线制变送器的接线区别:
二线制变送器由模块供电,四线制变送器需要外部独立供电。

模拟量读数不稳定或跳变的排查:
1. 检查信号电缆是否使用屏蔽双绞线且屏蔽层单端接地;
2. 检查信号电缆是否与动力电缆分开敷设,平行敷设距离应大于 20cm;
3. 在程序中增加软件滤波(平均值滤波);
4. 检查变送器供电是否稳定。
